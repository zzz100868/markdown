# Markdown 在线编辑器学习项目说明

## 项目定位

本项目是一个基于开源 Arya Markdown 编辑器进行二次开发和学习实践的前端项目。项目围绕 Markdown 写作、实时预览、本地文档管理和多格式导出等场景展开，重点练习 Vue3、Vite、Pinia、组件拆分、浏览器本地存储和第三方编辑器集成。

作为第一个前端综合项目，它更适合作为「学习型实习项目」展示：能够体现对前端工程结构、组件通信、状态管理和用户交互流程的理解，也保留了继续优化的空间。

## 技术栈

- Vue3：页面和组件开发
- Vite：项目构建和开发服务器
- Pinia：文档列表、当前文档和编辑内容状态管理
- Vue Router：主页、关于页和导出页路由管理
- Element Plus：按钮、弹窗、消息提示等基础交互组件
- Vditor：Markdown 编辑、实时预览、工具栏和内容渲染
- localStorage：浏览器端文档持久化
- html2canvas、html2pdf.js、html-to-docx、Reveal.js：PNG、PDF、Word 和 PPT 预览导出能力

## 核心功能

- Markdown 在线编辑与实时预览
- 多文档本地管理，支持新建、切换、重命名和删除
- 编辑内容自动保存到浏览器本地存储
- 本地 Markdown 文件导入
- 支持导出 PNG、PDF、Word 和 PPT 预览
- 移动端提供文档列表入口，方便在小屏设备上切换文档
- 上传图片、导出文件等关键操作增加异常提示

## 我负责和练习的内容

- 将项目整理为 Vue3 + Vite 的开发结构
- 使用 Pinia 管理文档数据和当前编辑内容
- 封装 localStorage 适配层，统一处理文档列表、文档内容和历史数据迁移
- 实现侧边栏文档管理交互，包括新建、选择、重命名和删除
- 接入 Vditor 编辑器，处理初始化内容、切换文档和防抖保存
- 完善导入、导出和图片上传场景下的错误提示
- 适配移动端文档入口，修复移动端无法打开文档列表的问题
- 清理学习项目中暂不维护的测试脚本和冗余分析文件，保持项目结构简洁

## 项目亮点

- 不是单页面静态展示，而是一个可以实际使用的编辑工具
- 覆盖了前端项目常见能力：路由、状态管理、组件通信、本地存储、第三方库集成和构建部署
- 多文档管理和自动保存逻辑接近真实产品中的草稿箱/笔记场景
- 导出功能涉及 DOM 渲染、截图、PDF 生成、Word 文档生成和 PPT 预览，能体现对浏览器能力边界的探索

## 简历表述建议

可以写成：

> Markdown 在线编辑器学习项目：基于 Vue3、Vite、Pinia 和 Vditor 实现在线 Markdown 编辑、多文档本地管理、自动保存、本地文件导入及 PDF/PNG/Word/PPT 导出预览等功能。负责项目结构整理、状态管理、本地存储适配、核心交互修复和移动端文档列表适配。

## 后续可优化方向

- 增加更完整的端到端测试，覆盖导入、编辑、切换文档和导出流程
